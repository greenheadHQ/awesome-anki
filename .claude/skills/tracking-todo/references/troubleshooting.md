# TODO 추적 트러블슈팅

TODO/로드맵 관리 중 발생하는 정합성 문제 대응 절차.

## 1) 문서와 실제 구현 상태가 다름

- 증상: `미구현 기능`에 적힌 항목이 이미 구현됨
- 점검:
  - 관련 경로를 `rg`로 검색해 구현 여부 확인
  - 최근 변경 이력은 `git log -- <path>`로 확인
- 조치:
  - 구현 완료 항목은 제거 또는 완료 섹션으로 이동
  - 구현 범위가 부분적이면 제한사항을 명시

## 2) 우선순위 기준이 팀 컨텍스트와 불일치

- 증상: 긴급 항목이 낮은 우선순위에 남아 있음
- 점검:
  - 사용자 최근 요청 패턴과 현재 장애/리스크 기준 비교
- 조치:
  - 높은/보통/낮은 우선순위를 재분류
  - 재분류 근거를 항목 설명에 한 줄로 남김

## 3) 기술 부채 항목이 중복 또는 모호함

- 증상: 유사 항목이 여러 줄로 분산되어 추적 불가
- 점검:
  - 중복 키워드(예: 테스트, 에러 핸들링)로 군집화
- 조치:
  - 중복 항목 통합
  - 각 항목에 대상 경로 또는 모듈명을 명시

## 4) Deprecated 예정 목록이 오래됨

- 증상: 제거 대상 파일이 이미 삭제되었거나 경로가 바뀜
- 점검:
  - `test -e <path>` 또는 `find`로 파일 존재 여부 확인
- 조치:
  - 이미 삭제된 항목은 기록 후 목록에서 제거
  - 경로 변경 시 최신 경로로 갱신
