# SuperMemo's Twenty Rules 적용

## 카드 길이 기준

| 타입 | 구성 | 기준 범위 | 최대 |
|------|------|-----------|------|
| Cloze | 전체 | 40~60자 | 80자 |
| Basic | Front (Q:) | 20~30자 | 40자 |
| Basic | Back (A:) | ~20자 | 30자 |

## 필수 원칙 6가지 (MUST)

### 1. Minimum Information
카드당 **한 가지 사실만** 포함. 여러 사실을 담으면 학습 효율 저하.

### 2. One Answer Only
하나의 답만 허용. 복수 답이 필요하면 개별 카드로 분리.

### 3. No Yes/No
Yes/No Cloze에는 **힌트 필수**.
```
❌ {{c1::있다}}
✅ {{c1::있다::있다 | 없다}}
```
Cloze Enhancer가 이진 패턴 자동 감지하여 힌트 추가.

### 4. Context-Free
중첩 맥락 태그 필수. 카드만 보고 주제 파악 가능해야 함.
```
[DNS > Record > A] A 레코드는 {{c1::IPv4 주소::IP 버전}}를 매핑한다.
```

### 5. No Enumerations
열거형은 개별 카드로 분리.
```
❌ DNS 레코드 유형: {{c1::A, AAAA, CNAME, MX, NS}}
✅ 각 레코드 유형별 독립 카드
```

### 6. No Example Trap
역방향 질문 금지.
```
❌ "TCP의 예시는?" → 답이 무한정
✅ "TCP는 {{c1::연결 지향::연결 | 비연결}} 프로토콜이다"
```

## Self-Correction 루프

1. 카드 생성 후 글자 수 검토
2. 상한선 초과 시 재작성 (더 간결하게)
3. 재작성 후에도 초과 시 추가 분할

## 부정형 질문 방지

```
❌ "DNS는 UDP를 사용하지 않는다" → {{c1::O/X}}
✅ "DNS 쿼리는 {{c1::UDP::TCP | UDP}} 프로토콜을 사용한다"
```
